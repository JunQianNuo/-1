# 问题二运行结果分析

> 分析对象：当前已完成的两次运行——`run_q2_smoke.py`（链路验证）和 `run_q2_single_search.py`（单重覆盖粗搜索）。两次运行均在总卫星数 $S=40$ 的条件下进行，**均未找到满足单重覆盖（$\min c \ge 1$）的可行方案**。

## 2. 核心发现：$S=40$ 下不存在可行方案

### 2.1 定性结论

两次运行评估了所有 $S=40$ 的因子对（$(M,N) = (1,40), (2,20), (4,10), (5,8), (8,5), (10,4), (20,2), (40,1)$），遍历了多种倾角、相位因子、$\Omega_0$ 和 $u_0$ 组合。**全部候选的最小覆盖重数 $c_{\min} = 0$**，即每个候选星座在仿真时段内都存在覆盖空洞。

### 2.2 定量指标（粗搜索最优候选）

最优候选参数如下：

| 参数 | 值 |
|:--|:--|
| 轨道面数 $M$ | 1 |
| 每面卫星数 $N$ | 40 |
| 相位因子 $F$ | 0 |
| 轨道倾角 $i$ | $53^\circ$ |
| $\Omega_0$ | $30^\circ$ |
| $u_0$ | $0^\circ$ |

| 覆盖指标 | 值 | 含义 |
|:--|:--|:--|
| $\mathcal C_1$（面积—时间单重覆盖率） | $15.0\%$ | 加权平均下仅 15% 的（网格点, 时刻）被至少 1 颗星覆盖 |
| $\mathcal C_2$（面积—时间二重覆盖率） | $0.0\%$ | 无任何二重覆盖 |
| 平均覆盖重数 | $0.15$ | 平均每个网格点仅有 0.15 颗卫星 |
| $c_{\min}$（整区最差覆盖） | **0** | ❌ 始终存在未被覆盖区域 |
| 最大覆盖间隙 | $22500\,\mathrm{s}$（$6.25\,\mathrm{h}$） | 覆盖完全缺失长达整个仿真窗口 |
| $\mathcal C_2^{\text{strict}}$（整区二重时间比例） | $0.0\%$ | 无任何时刻整区二重覆盖 |

### 2.3 所有候选的覆盖水平分布

从 2000 个候选的 CSV 数据来看：

- **$c_{\min} = 0$**：全部 2000 个候选均不满足单重覆盖硬约束
- **$\mathcal C_1$ 范围**：$3.2\% \sim 15.0\%$，最高的是 $M=1,N=40$ 的单轨道面方案
- **多轨道面方案（$M\ge2$）的 $\mathcal C_1$ 反而更低**（约 $8.4\%\sim9.5\%$），因为 $S=40$ 固定时，分到多个轨道面会减少每面卫星数，单面覆盖带变弱
- **$\mathcal C_2^{\text{strict}}$ 全为 $0$**：没有任何方案在任何时刻实现全区域二重覆盖

## 3. 物理原因分析

### 3.1 问题一结论的边界

问题一得到 $N_{\min}=40$ 的结论成立，但前提是：

> **单轨道面沿轨连续覆盖 + 纬度带目标（经度不受限）**

问题二的目标是**固定经纬度矩形** $(4^\circ\sim53^\circ\mathrm{N},\,73^\circ\sim135^\circ\mathrm{E})$。单轨道面在任一时刻只能覆盖一条窄经度带，要覆盖 $73^\circ\mathrm{E} \sim 135^\circ\mathrm{E}$（经度跨度 $62^\circ$）的整个区域，必须在不同时刻让卫星覆盖带扫过不同经度——但这是**时间序列覆盖**问题，不能靠增加同面卫星数解决。

### 3.2 面积下界佐证

目标区域面积与单星覆盖面积之比：

$$
\frac{A_D}{A_{\text{sat}}} \approx 40
$$

这 40 恰好是问题一的 $N_{\min}$，但它只是**瞬时覆盖全部区域**所需的最少卫星数的必要下界——要求 40 颗星**同时分布在不同经度位置**。而 $S=40$ 的单轨道面方案把全部 40 颗星集中在一条轨道上，无法同时覆盖不同经度，因此面积下界虽然是 40，但 40 颗同轨卫星远不够。

### 3.3 多轨道面反而更差的原因

固定 $S=40$ 时，$M$ 增大意味着 $N$ 减小。例如 $M=2,N=20$：

- 每面只有 20 颗星，问题一已经证明 $N<40$ 时同轨覆盖带存在沿轨空隙
- 两面之间经度间隔 $180^\circ$，但中国区域经度跨度仅 $62^\circ$，两面不可能同时覆盖目标区域
- 因此 $M=2$ 的覆盖性能反而比 $M=1$ 更差

## 5. 当前搜索的局限

| 局限 | 影响 | 后续处理 |
|:--|:--|:--|
| $S$ 仅搜到 40 | 未触及可行性边界 | 需递增 $S$ 继续搜索 |
| 空间步长 $6^\circ$ | 可能漏掉小范围覆盖空洞 | 候选方案用 $2^\circ$ 细网格复核 |
| 时间步长 $900\,\mathrm{s}$ | 可能漏掉短时覆盖空洞（卫星 $95\,\mathrm{min}$ 周期内仅 ~6 个采样点） | 候选方案用 $60\sim180\,\mathrm{s}$ 复核 |
| 倾角仅 $49^\circ\sim53^\circ$ | 题给允许 $40^\circ\sim60^\circ$ | 扩大倾角搜索范围 |
| $S$ 候选上限 2000 | $S=40$ 时枚举空间已耗尽，但更大 $S$ 时枚举空间更大 | 按递增规模搜索策略继续 |

## 6. 下一步行动

1. **扩大 $S$ 搜索范围**：按 $S=41,42,\ldots$ 递增搜索，首个出现 $c_{\min}\ge 1$ 的 $S$ 即为单重覆盖可行规模的初步估计。

2. **扩大倾角范围**：将倾角候选扩展到 $40^\circ\sim60^\circ$ 全范围，步长 $2^\circ$（粗搜）→ $0.5^\circ$（细搜）。

3. **细化相位搜索**：当前 $\Omega_0,u_0$ 步长 $30^\circ$，对多轨道面方案过于粗糙。粗搜阶段保持 $10^\circ\sim15^\circ$，候选方案附近局部加密。

4. **分阶段搜索**：
   - 阶段一（粗搜）：空间 $6^\circ$、时间 $900\,\mathrm{s}$、倾角步长 $2^\circ$，目标找到 $c_{\min}\ge 1$ 的大致 $S$ 区间
   - 阶段二（中搜）：空间 $3^\circ$、时间 $180\,\mathrm{s}$，对粗搜候选验证
   - 阶段三（细搜）：空间 $1^\circ$、时间 $60\,\mathrm{s}$，对最终候选精化

5. **单重可行方案确定后**，转入二重覆盖调整——在单重最优 $S_1$ 基础上逐步增加卫星，直至 $\mathcal C_2^{\text{strict}} \ge 95\%$。
