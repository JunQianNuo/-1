## 3. 确认的问题

### P1：初筛提前退出，候选排名退化为 \(t=0\)

实际入口传入：

```python
stop_if_margin_below=-0.02
```

评价器检测到当前时刻裕度低于阈值后立即 `break`。本次1824条记录全部只有1个评价时刻，因此所谓6小时初筛实际只比较了初始时刻。

负裕度不是候选全天最坏裕度排名的有效上界，所以该机制会改变Top-K候选选择。

修复：

```python
stop_if_margin_below=None
```

并增加完整时间步数断言。

严重度：高。

### P2：CEGIS初始活动集只覆盖6小时

原活动时间轴使用 `screen_duration_hours`，而分离器检查24小时。局部优化初始约束缺少其余18小时，只能依靠每轮加入一个反例弥补；本次仅2轮，难以表示全天约束。

修复：活动时间轴使用 `separation_duration_hours`。

严重度：高。

### P3：审计字段记录局部保留数，不是最终精修数

原字段：

```python
selected_for_refinement = len(selected)
```

写入时尚未执行全局 `keep_top_per_total` 截断。因此全部为1表示每个结构局部保留1个起点，不表示全部进入精修。

修复：

- `local_starts_retained` 记录局部保留数；
- 全局Top-K后回填真正的 `selected_for_refinement`。

严重度：中。

### P4：最长未覆盖时长多算一个时间步长

原方法用连续False样本数乘以中位步长。24小时、120秒、含首尾端点的721点网格在全天未覆盖时得到86520秒，超过真实区间86400秒。

修复：使用实际时间坐标计算未覆盖区间。

严重度：中。

### P5：自适应证书失败后仍停止搜索

原代码只要存在数值候选，证书无论返回 `covered`、`uncovered` 或 `inconclusive`，之后都会执行外层 `break`。此外失败的较小星数候选会滞留，妨碍验证后续更大星数候选。

修复：

- 分开保存首个数值候选、当前总星数候选和最终接受候选；
- 只有 `certificate.status == "covered"` 才接受；
- 失败或未决时继续搜索后续总星数；
- 保存全部证书历史。

严重度：高。

## 4. 修复后含义

修复后的程序仍是外层最小卫星数搜索：

\[
S^*=\min\{S:\exists\theta,\ C(x,t;S,\theta)\ge q,\ \forall x,t\}.
\]

但只有满足完整离散时域、CEGIS收敛，并在启用证书时返回 `covered` 的第一个总星数，才会被接受。

## 5. 旧结果的使用边界

旧结果可以证明：原预算下15个被精修候选没有达到数值可行标准。

旧结果不能证明：

- 86520秒是合理的24小时失效时长；

因此必须修复后重新运行。
