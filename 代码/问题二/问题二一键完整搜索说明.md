# 问题二一键完整搜索说明

## 1. 能做什么

`run_q2_full_pipeline.py` 自动执行：

1. `400、800、1200、1600` 锚点粗筛；
2. 找到首个数值可行锚点后，以100为步长缩小区间；
3. 以20为步长再次缩小；
4. 在最多60颗宽的区间内逐整数搜索；
5. 对首个数值候选及其前3个星数做高预算复核；
6. 对候选运行自适应连续时空验证；
7. 如果证书未通过，向上继续验证最多20个总星数；
8. 输出完整JSON和CSV报告；
9. 每个总星数独立保存，支持断点续跑。

## 2. 放置位置

把：

```text
run_q2_full_pipeline.py
run_q2_full_pipeline.bat
```

放到：

```text
D:\知识库\数学建模\第一次\代码\问题二
```

该目录中应已经存在修复后的：

```text
run_q2_optimized_search.py
verify_q2_actual_fix.py
```

## 3. 执行

双击或在终端运行：

```bat
run_q2_full_pipeline.bat
```

## 4. 输出

主结果：

```text
results_full_pipeline\q2_full_pipeline_result.json
```

所有运行汇总：

```text
results_full_pipeline\q2_full_pipeline_runs.csv
```

每个阶段、每个卫星数均有独立目录和日志，例如：

```text
results_full_pipeline\01_anchors\S0400
results_full_pipeline\04_integer\S0731
results_full_pipeline\06_certificate\S0735
```

程序中断后重新运行会复用已经完整生成的结果。

## 5. 可能的结束状态

- `certified_candidate_found`：找到通过连续验证的候选；
- `not_found_at_anchors`：400–1600锚点均未找到数值候选；
- `integer_stage_no_candidate`：强预算整数搜索未复现粗筛候选；
- `confirmation_failed`：候选在高预算复核中未复现；
- `numerical_but_not_certified`：存在数值候选，但连续验证未通过或未决。

## 6. 结论边界

该脚本给出的是：

> 当前Walker参数化、当前预算和当前验证器下，第一个被接受的计算方案。

它不是严格的全局最优证明。原因是固定卫星数下的连续参数优化属于非凸启发式搜索，而且不同总星数的Walker结构集合并不严格嵌套。
