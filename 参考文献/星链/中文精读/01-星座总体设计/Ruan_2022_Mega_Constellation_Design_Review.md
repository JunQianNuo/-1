# 阮永井 2022 · 巨型星座设计综述（精读解读）

> **一句话总结**：本文是一篇中文综述，系统梳理了低轨（LEO）巨型星座从"构型设计"（任务需求、覆盖特性、摄动补偿、备份策略四个角度）到"构型控制"（初始化、保持、重构、碰撞规避、离轨五个任务段）的全寿命方法与特点，指出传统小型星座的设计/控制方法在数千至上万颗量级下失效，需发展多学科优化、自主运行、分阶段部署与多级库存备份等新方法。

> **阅读提示**：原文为中文综述，本篇为**重组织的精读梳理**而非翻译。原 MinerU 解析稿存在大量 OCR 缺失（数字、年份、引用编号被吞），凡关键数据均以原文表格为准复核；正文中被吞掉的数字用"（原文缺，待核对）"标注。本文几乎不含数学公式，属定性综述，重点在**方法脉络与可复用建模思路**。

## 一、论文速览
- **作者/年份/出处**：阮永井、胡敏（通讯）、云朝明，航天工程大学（北京）；发表于《中国空间科学技术》（Chinese Space Science and Technology），2022 年，DOI: 10.16708/j.cnki.1000-758X.2022.0001。
- **解决的问题**：面向低轨巨型星座（数千至上万颗卫星），综述"构型设计"与"构型控制"两大主题的研究现状、方法与不足，并给出未来研究方向。
- **核心贡献**（4 条）：
  1. 汇总了 SpaceX（Starlink）、OneWeb、Telesat 三大国外星座及中国 GW（国网）、鸿雁、虹云等星座的轨道参数（含高度、倾角、轨道面数、每面卫星数），为建模提供真实构型数据。
  2. 从**任务需求、覆盖特性、摄动补偿、备份策略**四个角度系统归纳构型设计方法，并逐一指出其在"巨型"量级下的局限。
  3. 按卫星全寿命流程（初始化 → 保持 → 重构 → 碰撞规避 → 离轨）梳理构型控制方法。
  4. 提出五大未来方向：多学科优化、自主运行管理、安全性与离轨、分阶段部署、备份策略。
- **方法类型**：**综述（Review）**，不含原创算法或公式推导，属定性归纳 + 参数汇总。

## 二、研究背景与动机
- 低轨（LEO，轨道高度低、传输延时短、路径损耗小）成为卫星通信/遥感/导航增强的战略高地，被称为空间战略资源的"新战场"。
- 巨型星座（成百上千甚至上万颗卫星）的构型与系统性能之间**耦合关系复杂**，且卫星数量的量级跃升使传统面向数十颗卫星的设计/控制方法不再适用。
- 中国低轨巨型星座（GW/国网，两个子星座共约 1.3 万颗）已向 ITU 提交申请，但研究相对国外滞后，故亟需系统梳理设计与控制方法。
- **数模关联**：本文提供了"星链问题"最基础的**输入数据（真实星座参数表）**和**方法地图（设计+控制各环节都有哪些可选方法）**，适合作为总体设计的"选题定位"参考。

**关键观察**：巨型星座普遍采用 **Walker 型 / 混合倾角多层壳（shell）** 构型——用不同倾角覆盖不同纬度带、用不同高度分层，卫星总数由 `轨道面数 × 每面卫星数` 累加。这正是数模里描述星座规模的基本关系：

$$N = \sum_{k} P_k \cdot S_k$$

其中 $N$ 为星座卫星总数（颗），$P_k$ 为第 $k$ 层（子星座/壳）的轨道面数（个），$S_k$ 为第 $k$ 层每个轨道面内的卫星数（颗）。同一层内卫星在轨道面间均匀分布（Walker 构型），面内卫星等相位间隔 $360^\circ/S_k$。

### 3.2 针对任务需求的构型设计

不同任务（通信 / 遥感 / 导航增强 / 定位授时）对应不同设计目标：
- **通信**：追求全球覆盖与连通性，代表构型即 Walker/混合壳。
- **遥感 + 通信异构星座**：为降低系统响应时间，用"预定义设计变量范围 → 生成数千个异构配置 → 按性能指标调优"的优化框架寻优。
- **导航增强（LEO augmentation）**：核心指标是**精度衰减因子 DOP / PDOP**（几何精度因子）与**可见卫星数**。文献用组合星座使可见星数与 DOP 在全球范围**均匀分布**，并把设计建模为多目标优化问题，用**多目标粒子群（MOPSO）**求解，以 PDOP、可见星数、轨道高度为性能指标，用模糊集从帕累托解集选最优。

DOP 是导航星座设计的关键量，其含义（原文未给公式，此处补充通用定义供建模用）：

$$\text{PDOP} = \sqrt{\sigma_x^2 + \sigma_y^2 + \sigma_z^2}\,/\,\sigma_0$$

其中 $\sigma_x,\sigma_y,\sigma_z$ 为三维位置估计标准差（m），$\sigma_0$ 为伪距测量标准差（m），PDOP 无量纲；PDOP 越小表示卫星几何分布越好、定位精度越高。（原文只定性提到"使 DOP 分布均匀"，本式为通用定义，属补充。）

**局限（原文强调）**：现有导航/遥感星座优化多为**混合星座、卫星数 ≤ 数十颗**；卫星数上千后优化搜索空间暴涨几个数量级，**易陷局部最优、难得全局最优**，需在传统方法上探索新设计法。

### 3.3 针对覆盖特性的构型设计

- 覆盖性能是**多属性指标体系**（连续覆盖、单/多重覆盖、重访频率、区域 vs 全球）。
- 传统覆盖优化用于低轨侦察星座，**卫星数一般不超过数十颗，只能实现部分区域覆盖**，无法直接支撑持续全球覆盖的巨型星座。
- 面向巨型星座的代表方法：先建**快速而准确的轨道预报器**，再用**球面 Voronoi 镶嵌 + Delaunay 三角剖分**（以卫星星下点为顶点）来表征覆盖与连通性，配合重访频率、星间链路可行性指标，最后用**模拟退火（SA）**优化，输出最佳轨道高度、倾角、每面卫星数、轨道面数。该框架把星座设计直接与系统成本、可扩展性、有效性挂钩。

> 球面 Voronoi/Delaunay 是巨型星座覆盖建模的核心几何工具：以星下点为种子把地球表面剖分成"责任区"，覆盖空洞与星间链路拓扑可由剖分结构直接读出。

### 3.4 考虑摄动补偿的构型设计（数模总体设计的核心机理）

**摄动来源**：低轨卫星受主要摄动力包括——非球形引力摄动（以 $J_2$ 项为主）、日月三体引力摄动、大气阻力摄动、潮汐摄动、太阳光压摄动。文献对各摄动量级做了估计。

**核心思想——参数偏置补偿**：在几何构型设计阶段对轨道要素做**小量偏置（bias）**，用摄动力自身的规律去抵消长期漂移，从而提高构型长期稳定性。典型手段：
- 用参数偏置处理卫星摄动运动导致的**构型发散**；
- 对多种摄动的整体影响做**数据拟合**再统一调整，消除长期偏差；
- 对低轨 **Walker 星座**分析摄动与相对漂移特点，提出**"多次偏置控制"策略**，把相对漂移量降到很低（原文具体阈值缺）。

**升交点赤经漂移率补偿机理**（$J_2$ 摄动，本文多处引用的关键关系，原文未列式，此处补充通用式）：

$$\dot{\Omega} = -\frac{3}{2} J_2 \left(\frac{R_e}{p}\right)^2 n \cos i$$

其中 $\dot{\Omega}$ 为升交点赤经漂移率（rad/s），$J_2 \approx 1.0826\times10^{-3}$ 为地球扁率带谐系数（无量纲），$R_e$ 为地球赤道半径（km），$p = a(1-e^2)$ 为半通径（km，$a$ 半长轴、$e$ 偏心率），$n=\sqrt{\mu/a^3}$ 为平均角速度（rad/s，$\mu$ 地球引力常数），$i$ 为轨道倾角（°）。**工程含义**：只要各卫星的**平半长轴 $a$ 与平倾角 $i$ 保持一致**，各星 $\dot{\Omega}$ 就一致，轨道面相对关系不变，星座构型即可长期保持——这正是下文 3.6 中"二代铱星、Starlink 保持控制"的理论依据。

**局限**：偏置补偿只能补偿摄动长期变化中的**线性部分**，非线性部分难补；对**轨道高度低于某值（原文缺，通常指 ~500 km 以下）**、大气阻力主导的低轨巨型星座，补偿效果十分有限。

### 3.5 星座备份策略（本文最具建模价值的方向之一）

**传统三种备份模式**（表 5）：

| 备份模式 | 优点 | 缺点 |
|:--|:--|:--|
| 在轨备份（同面备份 In-plane） | 可用性高、可快速替换故障星、增强服务 | 需更多备份星、成本高 |
| 停泊轨道备份（Parking orbit） | 一颗可备多个轨道面、省燃料、灵活 | 替换时间长、成本较高、性能增强差 |
| 地面备份（Ground / 按需发射） | 按需发射、备份星最少 | 替换时间最长、性能退化最明显 |

传统备份的可靠性建模：基于**卫星可靠度、备份卫星可用性、固有可用度、平均修复时间（MTTR）**建立星座系统可靠度模型 + 备份星轨道设计约束 + 备份重构模型，一体化优化。

**面向巨型星座的新方法——多级库存（存储论）模型**：把星座备份看作**多层供应链系统**：
- 地面备份 = **供应商（supplier）**；
- 停泊轨道备份 = **仓库（warehouse）**；
- 在轨（工作轨道）备份 = **零售商（retailer）**。

通过设计一组低于星座轨道高度的停泊轨道作缓冲，用**多级库存（multi-echelon inventory）**优化公式确定各位置最佳备份数，在满足性能约束下**最小化系统维护成本**。更进一步，有文献用**马尔可夫决策过程（MDP）**建模：结合发射失败、硬件故障、碰撞风险等随机因素，计算星座内卫星的最佳部署/补充策略，最小化"与故障风险相关的总净成本"并保证系统级不间断服务。

> 这是把"库存管理/供应链"思想迁移到卫星备份的关键跨界方法，对数模里"如何配置备份卫星数量与位置以最小化成本"极有参考价值。

### 3.6 星座构型控制（按全寿命流程）

1. **初始化控制**：通过一系列变轨机动改变主/从星轨道要素形成目标构型；控制时间短、摄动长期影响不明显。方法涵盖两脉冲、三/四迹向脉冲初始化。工程实例：Starlink 由猎鹰 9 送入约 ~300 km 低轨（原文缺具体值），再用**星上氪离子（Kr）电推进**爬升进入预定轨道；铱星/OneWeb 也先入较低轨道再机动，且**改变半长轴时同时改变倾角**以保持 $\dot{\Omega}$ 漂移率不变（呼应 3.4 机理）。
2. **保持控制**：维持卫星站位、降低运维成本。关键手段是**调节半长轴**、以**升轨/降轨交替**维持相位与高度。Starlink 因轨道低、大气阻力大，平半长轴每天衰减约若干米（原文缺值），升/降轨控制频繁且部分降轨规律性差、振荡大。基于 **TLE（两行轨道根数）**反演可获得真实保持控制规律，供中国星座参考。
3. **重构控制**：星座由一种构型变到另一种（因性能升级、任务改变、卫星失效）。优化目标为**推进剂消耗、转移时间或两者组合**。方法包括：预置量机动 + 一阶梯度/邻近极值组合算法、基于相对轨道要素的燃料最优脉冲重构、基于**Lambert（朗伯）定理**的转移、混合入侵杂草+粒子群优化、多目标进化算法（分解型 MOEA/D，以全球平均覆盖率、燃料均衡性、重构总时间、总速度增量为指标构建帕累托前沿）。分阶段部署也是一种"渐进重构"。
4. **碰撞规避**：星座内/外碰撞风险。Starlink 早期发射 60 颗中约 3 颗失效（原文数字缺，为通用记载）成为潜在碎片。方法：近距离自主规避机动、径向/迹向分离、建立碰撞检测数学模型、通过调整**倾角与相位**增大轨道面交汇处的时间间隔以降低碰撞概率。
5. **离轨控制**：高低轨需主动离轨。方法：研究轨道参数—离轨代价—存留寿命关系、电离层阻力被动离轨、电推进主动离轨（用**增广拉格朗日粒子群 ALPSO**解小推力最优控制）、空间绳网捕获 + 充气增阻被动离轨。铱星实施了完整离轨计划作为工程范例。

## 四、关键结论与结果
- **核心判断**：现有面向**数十颗卫星**的小型星座设计与控制方法，**不能最优地设计/控制巨型星座**——优化搜索空间爆炸、传统备份不可扩展、频繁机动缩短寿命、传统重构破坏构型。
- **摄动主导变化**：低轨（尤其 <500 km）大气阻力主导，$J_2$ 偏置补偿只能补线性长期项，效果有限。
- **备份范式转变**：从"在轨/停泊/地面三选一"转向**多级库存 + MDP 随机优化**，把备份当供应链管理。
- **控制范式建议**：将**重构控制、保持控制、备份策略三者结合做综合优化**。
- **未来五方向**：①构型设计与控制的多学科优化；②自主运行管理（自主导航+自主构型维持）；③安全性与离轨技术（临界密度、碎片雪崩效应）；④分阶段部署（多目标优化平衡各阶段服务/代价/扩展性）；⑤备份策略（多级库存、库存管理、最优备份星数）。
- **适用范围与局限**：本文为定性综述，**不提供可直接套用的原创公式或算例**；数据表可用，但方法细节需回溯所引原文。

## 五、对数模题目的启示（总体设计）
- **可复用的参数/数据**：表 1/2/4 提供 Starlink、OneWeb、Telesat、GW 的真实高度、倾角、轨道面数、每面卫星数，可直接作为总体设计的构型基线或校验值。
- **可复用的建模关系**：
  - 星座规模 $N=\sum_k P_k S_k$（分层 Walker 求和）；
  - $J_2$ 摄动 $\dot{\Omega}$ 漂移式——用"平 $a$、平 $i$ 一致"作为**构型保持的等式约束**，是偏置补偿设计的抓手；
  - 覆盖建模用**球面 Voronoi + Delaunay 三角剖分**，把"全球覆盖/星间链路"转成几何剖分问题；
  - 备份建模用**多级库存/MDP**，目标函数取"系统维护/故障净成本最小"。
- **建模时如何用**：
  1. 若题目要"设计构型"→ 以 Walker/混合壳为决策变量（$h, i, P, S$），用多目标优化（MOPSO/SA/MOEA/D）在覆盖率、连通性、成本、稳定性间求帕累托前沿；
  2. 若题目要"构型保持/摄动"→ 用 $\dot{\Omega}$ 一致条件设偏置量，评估大气阻力下半长轴衰减导致的机动频次与寿命；
  3. 若题目要"可靠性/备份"→ 用多级库存把地面/停泊/在轨三级作供应链，最小化维护成本或用 MDP 求补充策略。
- **注意事项/局限**：
  - 本文是综述，**公式需回溯原始引文**（原文正文几乎无推导）；
  - 上千颗量级下优化**易陷局部最优**，需注意算法可扩展性；
  - 低轨大气阻力使摄动补偿效果有限，不可直接照搬中高轨方法；
  - 传统重构会破坏构型，巨型星座宜"重构+保持+备份"联合优化。
