# 问题三求解算法设计

> [!important] 本文件定位
> 本文件是 [[数学建模/第一次/问题分析/星链系统-文献驱动版/21-问题三参数化求解模型.md|21-问题三参数化求解模型]] 的后续：只给出**如何求解**，不代入任何具体星座方案，不给最终平均时延、最大时延或吞吐量数值。

## 1. 总体算法目标

对任意输入星座参数

$$
\mathcal X=(M,N,F,i,\Omega_0,u_0),
$$

求解下列对象：

1. 每个时刻的星间链路快照图 $G_k=(V,E_k)$；
2. 区域 OD 点对的最小时延路径；
3. 平均时延、最大时延、不可达率等评价指标；
4. 均匀业务需求下的节点/链路负载；
5. 容量受限时的多路径分流与拥塞控制方案。
6. 同时满足问题二覆盖和问题三双比例通信约束的最低成本离散构型。

算法整体结构为：

```text
星座参数
  -> 卫星轨道传播
  -> ISL 快照图生成
  -> 地面点接入集合计算
  -> 最小时延路由
  -> 时延指标统计
  -> OD 业务矩阵
  -> 最短路负载统计
  -> 多路径分流优化
  -> 吞吐量与拥塞指标
```

## 2. 输入与输出

### 2.1 输入

| 输入 | 记号 | 说明 |
|:--|:--|:--|
| 星座参数 | $\mathcal X=(M,N,F,i,\Omega_0,u_0)$ | 不固定具体数值，由问题二或情景分析给出 |
| 目标区域 | $D$ | 经纬度矩形或更一般区域 |
| 时间窗 | $[0,T_{\mathrm{sim}}]$ | 可取轨道周期、恒星日或自定义仿真窗 |
| 时间步长 | $\Delta t$ | 需做收敛检验 |
| 地面采样点 | $\mathcal G=\{G_1,\ldots,G_J\}$ | 区域内网格点或随机点 |
| OD 点对集合 | $\mathcal P$ | 可取全对、抽样对或代表城市对 |
| 链路距离阈值 | $L_{\max}$ | 题给 $5000\,\mathrm{km}$ |
| 星上处理时延 | $\tau_{\mathrm{proc}}$ | 题给 $0.5\,\mathrm{ms/跳}$ |
| 单星接入容量 | $C_s^{\mathrm{access}}$ | 题给 $20\,\mathrm{Gbps}$ |
| 时延上限 | $T_{\mathrm{lim}}$ | 题给 $30\,\mathrm{ms}$ |
| 业务总流量 | $F_{\mathrm{tot}}(t)$ | 峰值为平均值的 $1.5$ 倍 |

### 2.2 输出

| 输出 | 文件建议 | 说明 |
|:--|:--|:--|
| 拓扑快照统计 | `q3_topology_summary.csv` | 每时刻边数、最大度、连通分量、最大链路距离 |
| 接入统计 | `q3_access_summary.csv` | 每时刻各地面点可接入卫星数 |
| 时延样本 | `q3_delay_samples.csv` | 每个 OD、时刻的最小时延、跳数、路径 |
| 时延汇总 | `q3_delay_summary.csv` | 平均时延、最大时延、分位数、不可达率 |
| 基线负载 | `q3_load_baseline.csv` | 最短路承载下的节点/链路负载 |
| 多路径优化 | `q3_multipath_result.csv` | 分流后负载、利用率、吞吐量 |
| 图表 | `figures/q3_*.png` | 时延分布、拓扑变化、瓶颈热力图 |

> [!note] 输出口径
> 本文档只规定输出结构，不要求现在生成这些文件。

## 3. 算法 A：时间离散与仿真网格生成

### 3.1 时间离散

令

$$
t_k=k\Delta t,\qquad k=0,1,\ldots,K-1.
$$

其中

$$
K=\left\lceil \frac{T_{\mathrm{sim}}}{\Delta t}\right\rceil.
$$

### 3.2 时隙步长初选

初始 $\Delta t$ 可按以下原则选取：

1. 链路距离在时隙内变化小；
2. 不跨越显著链路通断事件；
3. 计算量可接受。

步长收敛条件：

$$
\max_{k}
\max_{(p,q)\in E_k}
\frac{
\left|d_{pq}(t_k+\Delta t)-d_{pq}(t_k)\right|
}{
d_{pq}(t_k)
}
\le\varepsilon_d.
$$

若不满足，则将 $\Delta t$ 减半重算。

### 3.3 地面点采样

目标区域 $D$ 可生成三类采样点：

| 点集 | 用途 |
|:--|:--|
| 均匀网格点 | 统计平均/最大时延 |
| 边界加密点 | 捕捉区域边缘接入风险 |
| 随机点 | 做蒙特卡洛稳健性验证 |

OD 点对集合 $\mathcal P$ 可取：

1. 全部有序点对；
2. 全部无序点对；
3. 分层抽样点对；
4. 代表城市/边界点对。

## 4. 算法 B：卫星位置传播

### 4.1 输入

$$
\mathcal X=(M,N,F,i,\Omega_0,u_0),\qquad t_k.
$$

### 4.2 输出

惯性系卫星位置矩阵：

$$
\mathbf R^I_k
=
\left[
\mathbf r^I_1(t_k),\ldots,\mathbf r^I_S(t_k)
\right].
$$

地固系卫星位置矩阵：

$$
\mathbf R^E_k
=
\left[
\mathbf r^E_1(t_k),\ldots,\mathbf r^E_S(t_k)
\right].
$$

### 4.3 伪代码

```text
Algorithm B: SatellitePosition(X, t_k)
Input: M,N,F,i,Omega0,u0,t_k
Output: R_I[k], R_E[k]

1. S <- M*N
2. for m = 0,...,M-1:
3.     Omega_m <- Omega0 + 2*pi*m/M
4.     for n = 0,...,N-1:
5.         u_mn <- u0 + 2*pi*n/N + 2*pi*F*m/(M*N) + n0*t_k
6.         r_I[m,n] <- R3(Omega_m) R1(i) [a cos u_mn, a sin u_mn, 0]^T
7.         r_E[m,n] <- R3(-omega_e*t_k) r_I[m,n]
8. return R_I, R_E
```

## 5. 算法 C：ISL 快照图生成

### 5.1 目标

在每个时刻 $t_k$ 生成：

$$
G_k=(V,E_k,W_k),
$$

其中 $W_k$ 是边权。

### 5.2 候选边生成

候选边包括：

1. 同轨前后相邻边；
2. 左右相邻轨道面最近卫星边。

同轨边可直接由索引生成。

邻轨边有两种实现：

#### 方法 C1：朴素最近搜索

对每颗卫星 $(m,n)$，在相邻轨道面 $m\pm1$ 的 $N$ 颗卫星中找最近者。

复杂度：

$$
O(MN^2).
$$

#### 方法 C2：规则 Walker 映射加速

对规则 Walker 构型，相邻轨道面最近星通常满足稳定的索引偏移：

$$
n^*_{\pm}\approx n+\Delta n_{\pm}\pmod N.
$$

可先在少数时刻求出 $\Delta n_{\pm}$，若稳定，则后续直接按索引偏移生成邻轨边。

复杂度：

$$
O(MN).
$$

#### 方法 C3：KD-tree 近邻搜索

在相邻轨道面卫星位置上构造 KD-tree，批量查询最近卫星。由于题面要求“相邻轨道面内最近卫星”，本题默认实现应采用该口径，而不是把邻轨边改写成自由拓扑优化。

复杂度约为：

$$
O(MN\log N).
$$

### 5.3 距离阈值与边权

对候选边 $(p,q)$：

1. 计算距离 $d_{pq}(t_k)$；
2. 若 $d_{pq}(t_k)>L_{\max}$，删除该边；
3. 若保留，则边权为

$$
w_{pq}(t_k)
=
\frac{d_{pq}(t_k)}{c}
+\tau_{\mathrm{proc}}.
$$

### 5.4 拓扑校验

每个快照必须输出以下诊断量：

| 诊断量 | 判定 |
|:--|:--|
| 最大度数 $\max_s \deg_s(t_k)$ | 应满足 $\le4$ |
| 最小度数 $\min_s \deg_s(t_k)$ | 检查是否出现孤立或半孤立卫星 |
| 连通分量数 | 若 $>1$，部分 OD 可能不可达 |
| 最大链路距离 | 应满足 $\le L_{\max}$ |
| 边数 $|E_k|$ | 检查拓扑是否异常跳变 |

### 5.5 伪代码

```text
Algorithm C: BuildISLGraph(R_I[k], X)
Input: satellite inertial positions R_I[k], constellation X
Output: weighted graph G_k

1. E_cand <- empty set
2. for each satellite (m,n):
3.     add intra-plane candidates (m,n)<->(m,n+1), (m,n)<->(m,n-1)
4.     find nearest satellite in plane m+1 and add candidate
5.     find nearest satellite in plane m-1 and add candidate
6. E_k <- empty set
7. for each candidate edge (p,q) in E_cand:
8.     d <- ||r_p^I(t_k)-r_q^I(t_k)||
9.     if d <= L_max:
10.        w <- d/c + tau_proc
11.        add edge (p,q,w) to E_k
12. compute degree, connected components, max distance
13. return G_k
```

## 6. 算法 D：地面点接入集合计算

### 6.1 目标

对每个地面点 $G_a$ 和时刻 $t_k$，求可接入卫星集合：

$$
\mathcal A_a(t_k)
=
\{s:\gamma_{s,a}(t_k)\le\theta\}.
$$

### 6.2 朴素算法

对每个地面点遍历所有卫星：

$$
O(JS).
$$

### 6.3 加速算法

将卫星地固系单位向量建立球面近邻索引，只查询角距离小于 $\theta$ 的卫星。

可选实现：

1. 3D KD-tree；
2. Ball tree；
3. 经纬度网格桶；
4. 直接使用问题二覆盖判定模块。

### 6.4 伪代码

```text
Algorithm D: AccessSet(R_E[k], G)
Input: satellite ECEF positions R_E[k], ground points G
Output: access sets A_a[k]

1. for each ground point G_a:
2.     A_a <- empty set
3.     for each satellite s:
4.         gamma <- central_angle(r_s^E, g_a)
5.         if gamma <= theta:
6.             add s to A_a
7.     record |A_a|
8. return all A_a
```

## 7. 算法 E：最小时延路由

### 7.1 直接枚举接入卫星对

对 OD 对 $(A,B)$：

$$
T_{AB}(t_k)
=
\min_{s\in\mathcal A_A,r\in\mathcal A_B}
\left[
\tau_{A\to s}
+D_k(s,r)
+\tau_{r\to B}
\right].
$$

直接枚举所有接入卫星对可行，但重复计算较多。

### 7.2 推荐算法：超级源多源 Dijkstra

对一个源地面点 $A$，构造超级源 $\sigma_A$：

1. $\sigma_A$ 与所有 $s\in\mathcal A_A$ 相连；
2. 边权为 $\tau_{A\to s}$；
3. 在图 $G_k$ 上运行一次多源 Dijkstra；
4. 对任意目的点 $B$，取

$$
T_{AB}(t_k)
=
\min_{r\in\mathcal A_B}
\left[
\operatorname{dist}_{\sigma_A}(r)
+\tau_{r\to B}
\right].
$$

这样每个时刻只需对每个源地面点运行一次 Dijkstra。

### 7.3 复杂度

设地面点数为 $J$，卫星数为 $S$，ISL 边数为 $E$，时刻数为 $K$。

每个快照的最短路复杂度约为：

$$
O\left(J(E+S)\log S\right).
$$

总复杂度为：

$$
O\left(KJ(E+S)\log S\right).
$$

因为每颗卫星最多 4 条 ISL，故

$$
E=O(S),
$$

所以可近似为：

$$
O(KJS\log S).
$$

### 7.4 伪代码

```text
Algorithm E: MinDelayRouting(G_k, AccessSets)
Input: graph G_k, access sets A_a[k]
Output: T_ab[k], path_ab[k]

1. for each source ground point a:
2.     sources <- empty list
3.     for each satellite s in A_a[k]:
4.         uplink_delay <- ||r_s^E - g_a|| / c
5.         add (s, uplink_delay) to sources
6.     dist, predecessor <- MultiSourceDijkstra(G_k, sources)
7.     for each destination ground point b:
8.         best_delay <- +infinity
9.         best_sat <- None
10.        for each satellite r in A_b[k]:
11.            downlink_delay <- ||r_r^E - g_b|| / c
12.            delay <- dist[r] + downlink_delay
13.            if delay < best_delay:
14.                best_delay <- delay
15.                best_sat <- r
16.        record T_ab[k] and reconstruct path if needed
17. return delay table and paths
```

## 8. 算法 F：时延指标统计

### 8.1 基础指标

对所有 OD 点对和时刻统计：

$$
\bar T
=
\operatorname{mean}(T_{ab,k}),
$$

$$
T_{\max}
=
\max(T_{ab,k}),
$$

$$
P_{\mathrm{unreach}}
=
\frac{\#\{T_{ab,k}=+\infty\}}{\#\{(a,b,k)\}}.
$$

### 8.2 分位数指标

为了避免单个异常采样点主导评价，可同时统计：

$$
T_{50},\quad T_{90},\quad T_{95},\quad T_{99}.
$$

### 8.3 分组指标

可按以下维度分组统计：

1. 时间段；
2. 源点纬度；
3. 目的点纬度；
4. OD 距离；
5. 接入跳数；
6. 路径经过面间链路数。

### 8.4 伪代码

```text
Algorithm F: DelayStatistics(T_table)
Input: T_ab[k]
Output: delay metrics

1. remove or separately mark unreachable samples
2. compute mean, max, median, p90, p95, p99
3. compute fraction T <= T_lim
4. compute unreachable ratio
5. group by time and geography if needed
6. return summary tables
```

## 9. 算法 G：最短路基线负载统计

### 9.1 目标

先不做优化分流，而是假设所有 OD 流量走最小时延路径，统计负载分布。

这一步用于回答：

1. 最短路策略是否会导致负载集中；
2. 哪些卫星/链路最容易成为瓶颈；
3. 后续多路径分流能改善多少。

### 9.2 OD 流量矩阵

均匀业务需求下：

$$
q_{ab}(t_k)
=
F_{\mathrm{tot}}(t_k)\omega_a\omega_b,
\qquad a\ne b.
$$

若排除自通信，需要归一化：

$$
q_{ab}(t_k)
=
\frac{F_{\mathrm{tot}}(t_k)\omega_a\omega_b}
{1-\sum_j\omega_j^2}.
$$

### 9.3 负载累加

对每个 OD 对的最短路径 $P_{ab}(t_k)$：

- 对路径经过的 ISL 边累加 $q_{ab}(t_k)$；
- 对路径经过的中继卫星累加 $q_{ab}(t_k)$；
- 对源/宿接入卫星累加接入流量。

### 9.4 伪代码

```text
Algorithm G: BaselineLoad(paths, q_ab)
Input: shortest paths P_ab[k], traffic q_ab[k]
Output: node loads, link loads

1. initialize R_access[s] = 0
2. initialize R_relay[s] = 0
3. initialize R_link[e] = 0
4. for each OD pair (a,b):
5.     flow <- q_ab[k]
6.     path <- P_ab[k]
7.     add flow to access satellites of a and b
8.     for each ISL edge e in path:
9.         R_link[e] += flow
10.    for each relay satellite s in path:
11.        R_relay[s] += flow
12. return load tables
```

## 10. 算法 H：候选多路径生成

### 10.1 目标

对每个 OD 对生成候选路径集合：

$$
\mathcal K_{ab}(t_k)
=
\{P_{ab}^{(1)},\ldots,P_{ab}^{(K_p)}\}.
$$

候选路径应满足：

1. 时延不过大；
2. 尽量链路或节点不重合；
3. 能绕开高负载区域。

### 10.2 方法 H1：$K$ 条最短路径

可用 Yen 算法或 Eppstein 算法生成 $K_p$ 条简单路径。

筛选条件：

$$
T_{ab}^{(r)}(t_k)\le T_{\mathrm{lim}}
$$

或

$$
T_{ab}^{(r)}(t_k)
\le
(1+\epsilon_P)T_{ab}^{(1)}(t_k).
$$

### 10.3 方法 H2：删边/删点重算

先求最短路径 $P^{(1)}$，再删除部分关键边或中继节点，重新运行 Dijkstra。

优点：实现简单，可得到近似不相交路径。

### 10.4 方法 H3：负载惩罚路径

把边权改为：

$$
w'_{e}
=
w_e
+\eta_{\rho}\rho_e,
$$

然后重新求最短路，使路径倾向于绕开高负载链路。

### 10.5 伪代码

```text
Algorithm H: CandidatePaths(G_k, OD pair)
Input: graph G_k, source access set, destination access set
Output: candidate path set K_ab

1. compute shortest path P1
2. K_ab <- {P1}
3. while |K_ab| < Kp:
4.     modify graph by deleting used edge/node or adding load penalty
5.     compute next path P
6.     if P exists and delay(P) <= threshold:
7.         add P to K_ab
8.     else:
9.         break
10. return K_ab
```

## 11. 算法 I：多路径流量分配优化

### 11.1 变量

对 OD 对 $(a,b)$ 的第 $r$ 条候选路径设置流量变量：

$$
f_{ab}^{(r)}(t_k)\ge0.
$$

### 11.2 约束

需求守恒：

$$
\sum_r f_{ab}^{(r)}(t_k)=q_{ab}(t_k).
$$

接入容量：

$$
R_s^{\mathrm{access}}(t_k)\le C_s^{\mathrm{access}}.
$$

若设 ISL 链路容量情景参数 $C_e$，则：

$$
R_e(t_k)\le C_e.
$$

若设中继处理容量 $C_s^{\mathrm{relay}}$，则：

$$
R_s^{\mathrm{relay}}(t_k)\le C_s^{\mathrm{relay}}.
$$

### 11.3 目标函数

主目标：最小化最大利用率。

$$
\min \rho_{\max}.
$$

约束：

$$
R_e(t_k)\le \rho_{\max}C_e,\qquad \forall e,
$$

$$
R_s^{\mathrm{relay}}(t_k)\le \rho_{\max}C_s^{\mathrm{relay}},\qquad \forall s.
$$

时延—负载联合目标：

$$
\min
\alpha\rho_{\max}
+
(1-\alpha)
\sum_{a,b,r}f_{ab}^{(r)}T_{ab}^{(r)}.
$$

### 11.4 求解器建议

该问题在候选路径集合固定后是线性规划：

| 场景 | 求解器 |
|:--|:--|
| 小规模验证 | `scipy.optimize.linprog` |
| 中等规模 | HiGHS |
| 大规模 | 分时刻分解、OD 抽样、列生成 |

### 11.5 伪代码

```text
Algorithm I: MultipathTrafficEngineering
Input: candidate paths K_ab, demand q_ab, capacities
Output: path flows f_ab^r, max utilization rho_max

1. create one variable f_ab^r for each candidate path
2. create variable rho_max
3. add demand conservation constraints
4. add access capacity constraints
5. if ISL capacity is assumed:
6.     add link utilization constraints
7. if relay capacity is assumed:
8.     add relay utilization constraints
9. choose objective:
       a) minimize rho_max
       b) minimize alpha*rho_max + delay cost
10. solve LP
11. return flows and utilization
```

## 12. 算法 J：吞吐量上界搜索

### 12.1 需求缩放

令业务需求放大因子为 $\lambda$：

$$
q_{ab}^{(\lambda)}(t_k)
=
\lambda q_{ab}(t_k).
$$

### 12.2 最大可承载流量

求最大 $\lambda$，使得容量和时延约束可行：

$$
\lambda^*(t_k)=
\max \lambda.
$$

则时刻 $t_k$ 的吞吐量为

$$
\Phi(t_k)
=
\lambda^*(t_k)\sum_{a,b}q_{ab}(t_k).
$$

### 12.3 求解方法

可用二分搜索：

```text
Algorithm J: ThroughputSearch
Input: base demand q_ab, capacities, candidate paths
Output: lambda_star

1. low <- 0
2. high <- lambda_upper
3. while high-low > tolerance:
4.     mid <- (low+high)/2
5.     scale all demands by mid
6.     solve multipath feasibility LP
7.     if feasible:
8.         low <- mid
9.     else:
10.        high <- mid
11. return low
```

其中 $\lambda_{\mathrm{upper}}$ 可由总接入容量粗略上界给出：

$$
\lambda_{\mathrm{upper}}
\le
\frac{
\sum_s C_s^{\mathrm{access}}
}{
\sum_{a,b}q_{ab}
}.
$$

## 13. 算法 K：参数敏感性分析

虽然本文档不代入最终构型，但求解程序应支持对参数扫描。

### 13.1 相位因子 $F$ 扫描

对候选 $F$：

1. 生成拓扑；
2. 求平均/最大时延；
3. 求平均跳数；
4. 求最大负载。

输出函数：

$$
\bar T(F),\quad T_{\max}(F),\quad \bar H(F),\quad \rho_{\max}(F).
$$

### 13.2 时间步长收敛

比较：

$$
\Delta t,\quad \frac{\Delta t}{2},\quad \frac{\Delta t}{4}.
$$

若指标变化满足：

$$
\frac{|\bar T_{\Delta t/2}-\bar T_{\Delta t}|}{\bar T_{\Delta t/2}}
\le \varepsilon_T,
$$

则认为时间离散足够。

### 13.3 OD 采样收敛

比较不同地面采样密度下的指标：

$$
J_1<J_2<J_3.
$$

若平均时延和分位数变化稳定，则采样足够。


## 14. 算法 L：问题二—问题三联合多保真分支定界

### 14.1 饱和停止目标

联合反推不再将 $30\,\mathrm{ms}$ 作为通过率硬约束，而采用**覆盖为唯一淘汰、$P_{30}^{\mathrm{all}}$ 边际衰减判停**的目标：

覆盖 $\mathcal C_1\ge0.999$ 且 $\mathcal C_2\ge0.95$ 为唯一淘汰条件。对覆盖可行且完成高保真通信评价的星数层，取其 $P_{30}^{\mathrm{all}}$ 最优值记为 $P^{\star}(S)$。若某 $S_i$ 在其后 $200$ 星窗口（$[S_i, S_i+200]$）内

$$
\max_{S\in[S_i,S_i+200]} P^{\star}(S)-P^{\star}(S_i)\le0.01
\quad\text{且}\quad
\frac{\max_{S\in[S_i,S_i+200]} P^{\star}(S)-P^{\star}(S_i)}{200/100}\le0.005,
$$

则 $S_i$ 即为 $P_{30}^{\mathrm{all}}$ 已达饱和的最小推荐星数——增加 200 颗星只能提 $\le1\%$ 的全样本达标率、边际已衰减。$P_{30}^{\mathrm{reach}}$ 和 $T_{\max,\rm reach}$ 为诊断指标，不作为淘汰条件。

有关此前准严格双比例约束的搜索（$P_{30}^{\mathrm{reach}}\ge0.999$ 且 $P_{30}^{\mathrm{all}}\ge0.95$ 为硬约束）与完整运行报告，见 [[数学建模/第一次/问题分析/星链系统-文献驱动版/问题三联合搜索运行报告.md|问题三联合搜索运行报告]]。

### 14.2 共享快照内核

对候选构型的每个时刻只传播一次卫星位置：

$$
\mathcal S_k
=
\left(
\mathbf r^{ECI}(t_k),
\mathbf r^{ECEF}(t_k),
G_k,
A_k
\right).
$$

$\mathbf r^{ECEF}$ 同时供覆盖计数和星地接入使用，$\mathbf r^{ECI}$ 供 ISL 建图使用。覆盖与通信不再分别重复传播轨道。

### 14.3 批量最短路

对每个地面点建立源副本 $G_j^+$ 和目的副本 $G_j^-$：

- $G_j^+$ 只有指向可接入卫星的上行边；
- $G_j^-$ 只有从可接入卫星进入的下行边；
- ISL 边保持双向。

在该有向增广稀疏图上，以所有 $G_j^+$ 为源一次调用批量 Dijkstra，提取 $G_a^+$ 到 $G_b^-$ 的距离矩阵。由于目的副本没有出边，算法不会把地面点错误用作中继节点。

### 14.4 双比例乐观上界

设母样本总权重为 $Q$，已处理权重为 $q$，已处理可达权重为 $r$，30 ms 达标权重为 $h$，则

$$
P_{30,\mathrm{upper}}^{\mathrm{all}}
=
\frac{h+Q-q}{Q},
$$

$$
P_{30,\mathrm{upper}}^{\mathrm{reach}}
=
\frac{h+Q-q}{r+Q-q}.
$$

任一上界低于对应阈值即可严格淘汰。低保真层必须使用高保真母网格总权重作为分母，不能用低保真子集自身的样本数。

### 14.5 候选状态机

| 状态 | 含义 |
|:--|:--|
| `active` | 已通过当前层，等待继续评价 |
| `deferred` | 代理表现较差，但没有严格不可行证据 |
| `rejected` | 必要条件或严格上界证明不可行 |
| `verified` | 已完成规定高保真评价 |
| `numerical_error` | 数值异常，不能当作不可行 |

只有非空星数层的全部候选均为 `rejected`，该层才能标记为 `infeasible`。存在 `deferred`、`active` 或 `numerical_error` 时只能标记为 `inconclusive`。

### 14.6 发现与核验

```text
Algorithm L: JointCoverageCommunicationSearch
Input: q2 cache, S_LB, S_max, parameter grids, mother grids
Output: lowest discrete feasible constellation or auditable inconclusive result

1. discovery:
2.     load and deduplicate q2 coverage-feasible cache
3.     evaluate low -> medium -> high fidelity
4.     obtain a feasible star-count upper bound if possible
5. certification:
6.     for realizable star-count layers S in ascending order:
7.         generate all configured (M,N,F,i,u0)
8.         apply structural necessary conditions
9.         evaluate strict bounds and nested fidelity levels
10.        process deferred candidates before concluding the layer
11.        if layer is fully rejected: continue
12.        if layer has verified feasible candidate and no lower unresolved layer:
13.            return feasible_discrete
14.        otherwise return inconclusive when budget ends
```

发现阶段得到的是可行上界，不能证明更小星数不可行；核验阶段才负责最低成本结论。

### 14.7 断点与确定性并行

候选构型作为进程级并行单元，父进程分配确定性序号并单独写入 JSONL/CSV。检查点摘要绑定模型常数、阈值、网格、搜索范围和代码模式；摘要不一致时禁止复用。工作进程内部的 BLAS/OpenMP 线程数限制为 1，避免进程与线程双重过载。

## 15. 各算法复杂度审查

> [!note] 复杂度口径
> 本节只审查算法规模，不代入具体星座构型或最终数值。复杂度默认以一个完整仿真周期为单位；若按快照流式处理，空间复杂度可不随 $K$ 线性增长。

### 15.1 符号约定

| 记号 | 含义 |
|:--|:--|
| $S=MN$ | 卫星总数 |
| $K$ | 时间快照数 |
| $J$ | 地面采样点数 |
| $P=\mathrm{card}(\mathcal P)$ | OD 点对数量；若全有序点对，则 $P=J(J-1)$ |
| $E$ | 单个快照中的 ISL 边数；4 邻接约束下 $E=O(S)$ |
| $\bar a$ | 每个地面点的平均可接入卫星数 |
| $\bar h$ | 平均路径跳数或路径边数 |
| $K_p$ | 每个 OD 保留的候选路径数 |
| $B_\lambda$ | 吞吐量二分搜索迭代次数 |
| $C_{\mathrm{LP}}$ | 单个快照 LP 求解复杂度，取决于变量数、约束数和求解器 |

> [!warning] 表格修正
> 上表中的 $P=\mathrm{card}(\mathcal P)$ 是 OD 集合大小，不是路径集合大小；后文路径候选数统一记为 $K_p$，避免与 OD 点对集合混淆。

### 15.2 单算法复杂度汇总

| 算法 | 单快照时间复杂度 | 全时段时间复杂度 | 主要空间复杂度 | 审查结论 |
|:--|:--|:--|:--|:--|
| A 时间离散与采样 | $O(K+J+P)$ 为生成索引级对象；若只生成当前快照则 $O(J+P)$ | $O(K+J+P)$ | $O(K+J+P)$；流式为 $O(J+P)$ | 若采用全 OD，$P=O(J^2)$ 是第一个规模风险点。 |
| B 卫星位置传播 | $O(S)$ | $O(KS)$ | 单快照 $O(S)$；全部缓存 $O(KS)$ | 线性复杂度，可接受；建议流式生成位置。 |
| C ISL 快照图生成 C1 朴素 | $O(MN^2)=O(SN)$ | $O(KSN)$ | $O(S+E)$ | 能用但不建议大规模使用。 |
| C ISL 快照图生成 C2 Walker 映射 | $O(S)$ | $O(KS)$ | $O(S+E)$ | 可作规则 Walker 对照或近似加速；若题面坚持邻轨最近口径，则不能替代最近邻规则。 |
| C ISL 快照图生成 C3 KD-tree | $O(S\log N)$ 量级 | $O(KS\log N)$ | $O(S+E)$ | 推荐主口径；保持题面“邻轨最近卫星”规则，同时避免朴素 $O(MN^2)$ 搜索。 |
| D 地面接入集合朴素 | $O(JS)$ | $O(KJS)$ | $O(J\bar a)$ | 当 $J$ 大时可能成为瓶颈。 |
| D 地面接入集合索引加速 | $O(S\log S+J(\log S+\bar a))$ | $O(K[S\log S+J(\log S+\bar a)])$ | $O(S+J\bar a)$ | 推荐；复杂度应计入索引构造与输出规模。 |
| E 最小时延路由 | $O(J_s(E+S)\log S+P\bar a)$；若重构路径再加 $O(P\bar h)$ | $O(K[J_s(E+S)\log S+P\bar a+P\bar h])$ | 距离数组 $O(S)$；路径表 $O(P\bar h)$；图邻接表 $O(S+E)$ | 该复杂度成立的前提是图用邻接表查询邻居；若 `neighbors()` 每次扫描全边集，Dijkstra 会额外退化到近似 $O(SE)$ 量级。 |
| F 时延指标统计 | 均值、最大值、不可达率 $O(P)$；分位数排序 $O(P\log P)$ | 全局排序 $O(KP\log(KP))$；逐时刻统计 $O(KP\log P)$ | $O(KP)$；流式统计可降至 $O(1)$ 或直方图空间 | 若只要均值和最大值，可线性扫描。 |
| G 最短路基线负载 | $O(S+E+P\bar h)$ | $O(K[S+E+P\bar h])$ | $O(S+E)$；若保存所有路径为 $O(P\bar h)$ | 复杂度由路径长度累加决定，依赖 E 是否保存路径。 |
| H 候选多路径 H2/H3 | 每个 OD 为 $O(K_p(E+S)\log S)$ | $O(KP K_p(E+S)\log S)$ | $O(PK_p\bar h)$ | 这是进入多路径优化后的主要瓶颈之一。 |
| H 候选多路径 H1 Yen | 最坏约 $O(K_p S(E+S\log S))$ 每 OD | $O(KP K_p S(E+S\log S))$ | $O(PK_p\bar h)$ | 精确 K 最短路较重，建议论文主口径用近似删边或负载惩罚法。 |
| I 多路径流量分配 LP | 建模 $O(PK_p\bar h)$，求解由 $C_{\mathrm{LP}}$ 决定 | $O(K[PK_p\bar h+C_{\mathrm{LP}}])$ | 稀疏约束矩阵 $O(PK_p\bar h)$ | 固定候选路径后是线性规划；复杂度应表述为求解器主导。 |
| J 吞吐量上界搜索 | $O(B_\lambda[PK_p\bar h+C_{\mathrm{LP}}])$ | $O(KB_\lambda[PK_p\bar h+C_{\mathrm{LP}}])$ | 同算法 I | 若候选路径固定，可复用路径集合，只重复 LP 可行性检查。 |
| K 参数敏感性分析 | 单个参数情景等于所选管线复杂度 | $O(R_{\mathrm{case}}\,C_{\mathrm{pipeline}})$ | 取决于是否缓存情景结果 | 参数扫描本质是外层循环，会线性放大总计算量。 |
| L 联合多保真搜索 | 单候选高保真渐近量级不变；低层由覆盖点积、通信下界和批量 SSSP 构成 | $O(N_LC_L+N_MC_M+N_HC_H)$，其中 $N_H\ll N_L$ 为设计目标 | 单快照流式 $O(S+E+J^2)$，另有检查点记录 | 加速来自共享传播、编译后端批量 Dijkstra 和减少高保真候选数，不虚构高保真渐近阶下降。 |

其中 $J_s$ 为参与路由的不同源地面点数量。若所有地面点都作为源，则 $J_s=J$；若只抽样一部分源点，则 $J_s<J$。

### 15.3 总体复杂度判断

若采用推荐配置：

1. ISL 用 Walker 映射加速，即算法 C2；
2. 接入集合用空间索引加速；
3. 路由使用有向增广图上的批量 Dijkstra；
4. 暂不做多路径 LP，只统计时延与最短路负载；

则在 $E=O(S)$ 下，主流程复杂度可写成

$$
O\left(
K\left[
S
+S\log S
+J(\log S+\bar a)
+J_sS\log S
+P\bar a
+P\bar h
\right]
\right).
$$

若所有地面点都作为源，且使用全 OD，则 $J_s=J$、$P=O(J^2)$，路由后的目的扫描与路径存储会随 $J^2$ 增长。因此大规模实验应优先采用：

1. 分层抽样 OD 点对；
2. 只保存关键路径或瓶颈路径，不保存全部路径；
3. 分时刻流式计算统计量；
4. 多路径优化只对高负载 OD 或代表性 OD 执行。

若进入多路径分流与吞吐量搜索，总复杂度增加为

$$
O\left(
KPK_pS\log S
+
KB_\lambda C_{\mathrm{LP}}
\right),
$$

其中第一项来自候选路径生成，第二项来自多次 LP 可行性检查。该阶段通常比单纯时延统计更耗时，应作为第二阶段求解，而不是一开始对所有 OD、所有时刻全部展开。

### 15.4 复杂度审查结论

1. **算法 B、C2 是线性的**，不是主要瓶颈。
2. **算法 D 需避免 $O(KJS)$ 朴素全扫描**，建议使用 KD-tree、Ball tree 或复用问题二覆盖模块。
3. **算法 E 原有复杂度表述需要补充**：多源 Dijkstra 是主体，但 OD 目的接入扫描和路径重构在全 OD 时不可忽略。
4. **算法 H、I、J 是优化阶段瓶颈**，建议先完成时延统计与基线负载，再对瓶颈时刻/瓶颈 OD 做多路径优化。
5. **联合搜索的主要收益是减少高保真调用**：若低、中、高层候选数为 $N_L,N_M,N_H$，总成本为 $N_LC_L+N_MC_M+N_HC_H$，应通过阶段耗时和晋级率实测，而不是只给理论加速倍数。
6. **总体建模可行**：在 4 邻接 ISL 下 $E=O(S)$，图算法规模主要由 $K$、$J$、$P$ 和 $K_p$ 控制；通过共享快照、批量稀疏图计算和流式统计可以保持可计算性。

### 16.3 最适合在论文中重点强调的典型算法组合

问题三的主线可以概括为以下组合：

1. **动态快照图模型**：把时变星间网络离散成 $G_k=(V,E_k,W_k)$；
2. **空间近邻与可见性判定**：用角距离阈值和空间索引获得接入集合；
3. **多源 Dijkstra 最短路**：求每个时刻、每个源点到全网的最小时延；
4. **最短路基线负载统计**：识别无分流条件下的瓶颈链路和瓶颈卫星；
5. **K 最短路或启发式候选路径生成**：为多路径分流提供路径集合；
6. **线性规划流量工程**：在容量约束下最小化最大利用率；
7. **二分搜索吞吐量上界**：通过需求缩放和 LP 可行性判断求最大承载能力。

### 16.4 论文措辞建议

推荐写法：

> 本文采用动态快照图模型描述 LEO 星座网络的时变拓扑，并结合空间索引方法加速可接入卫星集合计算。在每个快照中，构造超级源并采用多源 Dijkstra 算法求解区域 OD 点对的最小时延路径；随后以最短路全量分配作为基线负载模型，识别网络拥塞瓶颈。进一步地，利用 K 最短路径或负载惩罚启发式生成候选多路径集合，并在候选路径固定后建立线性规划模型，以最小化最大链路/节点利用率。最后，通过二分搜索需求缩放因子，估计容量约束下的吞吐量上界。

不建议写法：

1. 不宜说“本文提出一种全新的路由算法”；更准确是“组合典型图论算法和流量工程优化方法”。
2. 不宜把 Walker 映射加速说成通用算法；它只是利用星座规则结构的工程加速。
3. 不宜把删边/负载惩罚法说成严格最优算法；应称为候选路径生成启发式。

## 17. 验证与错误检查

### 17.1 拓扑验证

| 检查项 | 合格条件 |
|:--|:--|
| 最大度数 | $\max_s\deg_s\le4$ |
| 链路距离 | 所有保留边满足 $d_{pq}\le L_{\max}$ |
| 边权非负 | $w_e>0$ |
| 连通性 | 连通分量数应记录；若不连通，允许但需解释不可达 |
| 边数跳变 | 若边数突然变化，检查是否为真实断链 |

### 17.2 接入验证

| 检查项 | 说明 |
|:--|:--|
| 空接入点 | 若 $\mathcal A_G=\varnothing$，说明覆盖或采样存在风险 |
| 接入卫星数分布 | 用于判断多接入选择空间 |
| 星地距离范围 | 检查单位与坐标转换错误 |

### 17.3 路由验证

| 检查项 | 说明 |
|:--|:--|
| $A=B$ 近距离通信 | 时延应接近上行+下行或很小 ISL 段 |
| 对称性 | 无负载、无方向限制时 $T_{AB}\approx T_{BA}$ |
| Dijkstra 路径 | 路径边必须全部存在于 $E_k$ |
| 不可达比例 | 不可达应能追溯到无接入或图不连通 |

### 17.4 负载验证

| 检查项 | 说明 |
|:--|:--|
| 流量守恒 | 路径流量总和等于 OD 需求 |
| 接入容量 | 不得超过 $20\,\mathrm{Gbps}$ 主口径 |
| 相对负载 | 无 ISL 容量时只报告相对拥塞 |
| LP 可行性 | 不可行时区分接入不足、链路不足、时延约束过严 |

## 20. 论文中的算法表述

可写为：

> 本文采用“共享快照—批量最短路—多保真分支定界”的联合反推算法。每个时刻只传播一次卫星位置，并由问题二覆盖计数与问题三星地接入共同复用；在有向增广稀疏图上以所有地面源副本批量运行 Dijkstra，得到区域 OD 时延矩阵。搜索先复用问题二缓存快速发现联合可行星数上界，再从可靠下界按可实现星数逐层核验。低保真结果只负责排序，数学必要条件和严格比例上界才允许永久淘汰，从而在降低高保真调用数量的同时保持结论口径可审计。
